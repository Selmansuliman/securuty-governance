# Secure Development Policy

## Purpose
Ensure software is developed and deployed using secure practices and controls.

## Scope
All development teams and codebases, internal and outsourced.

## Requirements
1. Adopt Secure SDLC with defined gates (design, code, test, release).
2. Mandatory static (SAST) and dynamic (DAST) testing.
3. Code reviews and peer approval for production changes.
4. Use of secret scanning, dependency scanning and CI/CD security checks.
5. Maintain separation of environments (dev, test, prod).
