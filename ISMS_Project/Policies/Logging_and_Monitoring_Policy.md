# Logging & Monitoring Policy

## Purpose
Ensure logging, monitoring and alerting enable detection and forensic investigation of security events.

## Scope
All critical systems, security appliances, network devices, and applications.

## Requirements
1. Centralized log collection (SIEM).
2. Time-synchronization (NTP) across systems.
3. Retention: minimum 12 months for security logs (adjust per legal requirements).
4. Define alert thresholds and escalation paths.

## Evidence
- SIEM dashboards
- Retention policies
- Alerting playbooks

## References
ISO: A.8.15, A.8.16
